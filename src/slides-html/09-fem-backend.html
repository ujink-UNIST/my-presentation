CAIR stands for Computational Analysis Intermediate Representation. In this project, it is a Jinja2/YAML-style intermediate format that defines the analysis macro and execution recipe. The purpose is to describe how the analysis should run once, then lower the same representation to multiple solver backends such as APDL, FEniCS/Dolfinx, or custom solvers.
